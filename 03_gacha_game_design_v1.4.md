# 戦国経済圏OS ガチャ本体 ゲームデザイン案 Ver.1.0

作成日: 2026年7月6日
位置づけ: 戦国パスポート内「ガチャ」機能のゲーム設計アイデア整理。次のMVP機能仕様書へのインプット資料。

---

## 1. コアループの再設計

現行案(ランク:足軽→侍→武将→軍師→奉行→大名→将軍、ガチャ・ビンゴ・ミッション)に、**「天下統一」という明確なゴール**を持たせるメタゲーム構造を追加する。

### 国盗りメタゲーム(案)

- 戦国風にデフォルメした日本地図を66国程度に分割
- 各武将カードは特定の「国」に紐づく
- 対応武将を一定数集めると、その国を「制圧」できる
- 全国制圧(または一定割合)で「天下統一」達成 → 特別NFT・実績称号・限定特典を付与
- 制圧状況を地図上に可視化(何国制圧したか一目でわかる)

**狙い**: ガチャで武将を集める行為に、明確なゴールと途中経過の可視化を与える。7章で「後回し機能」とされている全国マップは、単純な塗り絵的UIであれば軽量版としてMVPに組み込む価値がある。

## 2. コレクション設計

| 要素 | アイデア |
|---|---|
| レアリティ | 足軽級/侍級/武将級/軍師級/大名級の5段階。上位ほど「国盗り」への貢献度が高い |
| 武将の個性 | 絵柄だけでなく、各武将に短い逸話・ステータス(統率/知略/勇猛など)を持たせ、集める理由に物語性を持たせる |
| コンプ要素 | 史実グルーピングでのセット収集(例:織田家臣団、武田二十四将)。コンプ特典は換金性のないバッジ・称号にする(法務方針と整合) |
| 被り救済 | 同じ武将が被った場合、上位武将への合成/昇格素材にする(「訓練」「登用」演出)。無駄感を減らし、有料ガチャへの納得感を上げる |

## 3. 継続させる施策(ガチャ以外の日課)

- **登城(ログインボーナス)**: 毎日ログインで「兵糧」(石高とは別の少量ボーナス通貨)を付与。7日/30日で累積ボーナス
- **合戦イベント(期間限定)**: 週替わり/月替わりで「〇〇の戦い」を開催。対応武将の排出率アップ+専用ビンゴを併設
- **軍団(ギルド的要素)**: 評議員・代理店の「家臣団」という位置づけを活用し、複数ユーザーでチームを組んで合戦イベントの達成度を競う。代理店の紹介活動が「家臣を集める」ゲーム内体験と直結する設計
- **番付(ランキング)**: 制圧国数・戦功でのランキングを月次リセットし、常に追いつけるチャンスを残す

## 4. 収益化ポイントとの接続

- 有料ガチャ限定武将は「制圧効率が良い(必要枚数が少ない)」という設計にし、無料勢との差別化を「早く天下統一に近づけるか」という体験価値に置く(換金性を匂わせない)
- 天下統一パス加入者向け特典は「1日の無料ガチャ回数増加」「合戦イベント参加権」など、時短・体験拡張型にする(景品規制と距離を置きやすい)

## 5. 独自性・差別化の視点

一般的なソシャゲのガチャと異なり、戦国経済圏OSの武将カードは**AIアート教室・NFTマーケットへの入口**でもある。

- 集めた武将をAIアートで自分なりにアレンジできる
- 上位武将はNFT化して手元に残せる

このゲーム外(既存事業)への接続が独自性となる。既存資料の「AIアート教室は作品投稿・NFT化の入口」という位置づけとも整合する。

## 6. 国盗りメタゲーム 詳細設計(確定分)

### 国とデータ構造
- 全66国。史実の令制国ベースを想定
- 1国あたり3武将を割り当てる(初期リリースは20〜30国分のコンテンツから投入、DB設計は66国分を最初から確保)

```
provinces(国)
- id, name, region, status(open/locked)

warlords(武将)
- id, name, province_id, rarity(足軽級/侍級/武将級/軍師級/大名級), stats, lore

user_warlords(所持武将)
- user_id, warlord_id, count, acquired_at

user_provinces(制圧状況)
- user_id, province_id, is_conquered, conquered_at
```

### 国盗りの方式:ランダム蓄積型(ビンゴ方式)を採用
- ユーザーが国を選んで挑戦する方式(状態遷移が必要)ではなく、**引いたカードが自然に蓄積し、揃った国から順に制圧される方式**を採用
- 既存の「ビンゴ」機能(7章)のロジックをそのまま国盗りに転用できるため、単独開発の負荷が小さい
- ハズレ落胆度が低い(何を引いても、どこかの国の進捗になる)

### 1国あたりの武将スロット構成
| スロット | 役割 | レアリティ目安 |
|---|---|---|
| 1体目 | コモン枠(出やすい) | 足軽級/侍級 |
| 2体目 | 中間枠 | 武将級 |
| 3体目 | レア枠(出にくい・その国の顔) | 軍師級/大名級 |

3体すべて揃うとその国が制圧される。

### 排出率:ユーザー進捗連動(制圧済み国数に応じて動的に難化)

| 制圧済み国数 | 3体目(レア枠) | 2体目(中間枠) | 1体目(コモン枠) |
|---|---|---|---|
| 0〜5国 | 15% | 30% | 55% |
| 6〜15国 | 10% | 30% | 60% |
| 16〜30国 | 6% | 28% | 66% |
| 31〜50国 | 3% | 25% | 72% |
| 51〜66国 | 1.5% | 20% | 78.5% |

- 国ごとに固定の難易度を設定する必要はなく、ユーザーの `conquered_provinces_count` を参照して排出率を都度計算する
- 制圧済みの国の武将が重複して出た場合は、上位武将への合成素材として活用する(被り救済ロジックと接続。無駄にならないため排出プールを絞り込む必要がなくシンプル)

### ガチャの回数設計

```
無料ガチャ: 1日1回(連続ログインボーナスで追加ガチャ権を付与)
有料ガチャ: 石高/ガチャ券消費で追加抽選。排出率は無料と完全に共通、回数のみ増える(1日上限あり)
```

- 排出率を有料側だけ優遇しないことで、「お金を払えば当たりやすくなる」という構造を避け、13章の方針(換金性・射幸性を煽らない)と整合させる
- 序盤(排出率15%)・1日1回のペースでは、最初の1国(3体)がおよそ1〜2週間で揃う想定

## 7. コンプ特典・天下統一達成特典(確定)

### 階層構造
```
① 国制圧時(小報酬): 石高ボーナス、制圧マーク表示、専用ミッション解放
② 地方コンプ(中間報酬・8〜10地方単位): 中間称号、限定ビンゴ/ガチャイベント解放
③ 天下統一達成(66国制圧=美濃国制圧): 
   - 称号「天下人」付与(パスポート上に表示)
   - 代表武将を1体選択 → その武将ベースの専用フレーム/エフェクトを合成した
     天下統一記念NFTを生成(絵柄は達成者ごとに異なる)
   - AIアート教室・NFTマーケットへの特別割引クーポン
   - 代理店ボーナスポイント(要検討事項として残す)
```

- 天下統一記念NFTは「達成時に代表武将を選ばせ、それをベースに画像を合成生成」する方式(選択制)。既存のガチャ用武将素材を流用できるため新規イラスト制作コストを抑えられる。

## 8. 岐阜(美濃国)を「最後に挑戦する国」に設定(確定)

- 美濃国は66国のうち**最後に解放される特別な国**として設計する
- **解放条件: 他65国のうち60国以上を制圧すると挑戦可能になる**(完全制覇待ちではなく、終盤で「岐阜に王手をかけている」体感期間を作る)
- 美濃国の3武将(斎藤道三・織田信長など)はレア度最高クラス(大名級の中でも最高峰、専用特別枠)
- 美濃国制圧 = 事実上の天下統一達成。60国達成時点で「岐阜への挑戦権解放」の特別演出を出す
- 史実的背景: 織田信長が美濃を制圧後「岐阜」と改名し「天下布武」を掲げた地。ゲームの天下統一というテーマと物語上一致する
- 運営面: 岐阜のスポンサー企業・地域コラボと接続しやすい(6章スポンサー収益との連動候補)

```
provinces テーブルに追加:
- unlock_condition(解放条件。例:「他65国のうち60国以上を制圧」)
- is_final_province(最終国フラグ。美濃国のみtrue)
```

## 9. ガチャ回数上限(確定)

```
通常時: 無料1回 + 有料最大3回 = 合計4回
イベント日: 無料上限・有料上限ともに管理画面から個別に引き上げ可能
  (例: 無料3回+有料5回=合計8回。数値は都度設定)
```

- 排出率は無料・有料で共通(前述)。上限のみ有料側で拡張できる
- イベント時は無料枠も同時に引き上げることで、「有料だけ優遇しない」という法務方針と一貫させる
- 管理画面に「本日の無料上限」「本日の有料上限」の設定項目を追加するのみで実装可能(コード修正不要の運用設計)

```
gacha_config テーブル:
- base_daily_free_limit(デフォルト1)
- base_daily_paid_limit(デフォルト3)
- event_free_limit_override
- event_paid_limit_override
- event_start_at, event_end_at
```

## 14. 代理店ボーナスポイントの扱い(確定:特典としては作らない)

- 天下統一達成特典は**代理店かどうかに関わらず全ユーザー共通**とする(称号・NFT・クーポンのみ)
- 理由: 代理店ランクは「累計売上」で評価する適法運用中の制度であり、そこに「ゲーム内実績」という売上と無関係な評価軸を混ぜると、評価基準が曖昧になり法務的な説明が難しくなる。長期実績(天下統一)を代理店報酬に絡めると昇格ロジックとも噛み合いが悪い
- ただし**紐付けの記録は残す**: どの代理店経由のユーザーが天下統一を達成したかは、売上集計(`agent_sales`)とは別に、参考データとして記録する

```
achievements(実績イベント)
- user_id, achievement_type(例: "tenka_toitsu")
- referring_agent_id(紐付け先代理店。売上とは切り離し参考データとしてのみ保持)
- achieved_at
```

将来的に「濃いファンを紹介できている代理店」を見分ける参考指標として活用できる余地を残す。

## 17. 戦国メタバースとの接続方針(確定)

### ガチャの位置づけの再定義
- ガチャは単なる継続接点ではなく、**戦国メタバース(将来構想)の世界観を体験させる予告編、および開発資金源**という位置づけを持つ
- そのため、ガチャ・国盗りのグラフィック品質は「最初は軽量でよい」という前回方針から、**優先的に投資すべき領域**に格上げする

### グラフィック方針
- 武将カードのイラストは、単なる紋章・アイコンではなく、**その武将がいる城下町の情景を感じさせる1枚絵**(シネマティック・実写風の戦国世界観)にする
- 制圧演出に、その国の城下町コンセプトアートをフェードインさせ、「この城下町は将来、戦国メタバースで実際に歩けるようになります」という一文を添える
- 一貫したアートスタイル(桜・城・石畳・夕景等のトーン)をスタイルガイドとして固定し、AI画像生成を使って75武将+66国分の素材を統一感を保ちつつ量産する(単独開発でのコスト・工数対策)

### ガチャ当たり演出
- リアルタイム3Dではなく、**事前レンダリングされた動画/Lottieアニメーションを再生する形式**を採用する(LIFF内WebViewの処理負荷を回避しつつ見栄えを確保)
- 光の演出・カード浮遊・背景に城下町の一部が見える等、映像的な当たり演出を検討

### メタバース本体との関係(将来構想・現時点では未着手)
```
戦国パスポート(LINE/LIFF) ← 今作っているもの。日常の接点・ガチャ・国盗り・資金調達
        ↓(将来)
戦国メタバース(独立Webアプリ・3D城下町、VRは不採用) ← 将来構想。ガチャ収益を開発資金に充当
```
- VRヘッドセットは不採用。ブラウザ上で歩き回れる3D空間(ウォークスルー型)を想定
- 3D城下町はLIFF(LINE内ブラウザ)では処理負荷上厳しいため、**メタバース本体は将来的にLINEの外の独立Webアプリとして切り出す**想定を置いておく(現時点では設計のみ、着手はしない)

### 今のうちに用意しておくデータ(将来の手戻り防止)
```sql
provinces テーブルに追加:
- landmark_name (text, nullable)         -- 将来の3D空間内シンボル建造物名(例:岐阜城)
- theme_description (text, nullable)     -- 将来の3D空間演出用の世界観設定文
- has_castle_town (boolean, default false)  -- 将来、城下町3D空間を持つ国かどうかのフラグ
```
- 66国すべてを3D化するのは非現実的なため、`has_castle_town` フラグで対象国を限定管理できるようにしておく(使い方は将来決定)

### 資金源としての訴求における注意(法務方針の維持)
- 「ガチャ収益がメタバース開発に使われる」という伝え方自体は問題ない
- ただし「投資すれば将来メタバースの価値が上がって得をする」という表現は使わない。あくまで「応援・体験の前払い」という文脈に留める(13章の射幸性・投資的表現の回避方針と一貫)
