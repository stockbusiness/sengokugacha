# 戦国経済圏OS プロジェクト整理ドキュメント Ver.1.0 への追加検討事項

作成日: 2026年7月6日
位置づけ: Ver.1.0本体への追補。開発は田中氏単独で行う前提。

---

## 1. 前提条件の更新

- **開発体制**: 田中氏が単独で開発する(Verselix委託ではない)
- **代理店の存在**: 既存プロジェクトには代理店が存在し、ランク制度あり
  - ランク: アドバイザー → ディレクター → エージェント
  - 昇格条件: 累計売上(自己購入+紹介経由の購入合計)。売上がないと昇格不可
  - 自己購入も売上としてカウントする
  - 報酬率はランクにより変動する
  - **既に法律内で適法運用中の制度**(本ドキュメントでは法的判断には立ち入らない)

---

## 2. 既存の紐付け・販売フローの整理

現在稼働している紐付けは以下の通り、**問い合わせベース**であり即時決済ではない。

```
LP(代理店ごとに紐付け)→ 問い合わせ → 人による説明 → 販売(クロージング)
```

- 対象: AIアート教室、マーケットプレイス(開発中)
- 紐付け機能はあるが、**報酬の自動カウント機能はまだない**
- 現状、Excelでの手動管理は「不明」(整理されていない可能性あり)

## 3. 戦国経済圏OSにおける「二つの入り口」

既存フロー(問い合わせ型)とは別に、**直接参加型の入り口**を新設する。

```
【既存フロー】LP → 問い合わせ → 説明 → 販売(高単価・要説明商品向け)
【新設フロー】戦国パスポート登録 → ガチャ(無料/有料) → 興味喚起 → 購入(セルフサーブ)
```

- ガチャは「敷居を下げる無料体験」と「有料課金による収益化」の両方を担う
- 代理店の紐付けは、既存LPと同じ発想で「紹介リンク経由の戦国パスポート登録」に紐付ける
  - 紹介リンク経由 → 登録時に代理店IDを記録
  - 直接登録 → 代理店紐付けなし(直販扱い)

---

## 4. NFT・ガチャの二次流通に関する方針

- **二次流通(公式によるマーケットプレイス機能)は想定しない**
- ユーザーが外部プラットフォームで個人的に売買することは、ポケモンカードと同様「公式が関与しない私的取引」として位置づける
- ガードレール:
  - LP・マーケティング表現で「値上がり」「投資」「稼げる」を一切使わない(体験・コレクション訴求のみ)
  - 代理店の営業トークにも同ガードレールを周知する必要あり
  - 利用規約に「換金性・転売価値の非保証」「二次流通への運営不関与」を明記
- **この整理自体、実際の適法性は専門家確認が必要な領域**

## 5. 開発方針への反映(単独開発前提)

### 代理店機能の段階的実装

| フェーズ | 内容 |
|---|---|
| Phase1(MVP) | 紹介リンク発行、購入・紹介イベントのDB記録のみ(ランク判定はまだしない) |
| Phase2 | 累計売上の自動集計・管理画面表示 |
| Phase3 | ランク自動昇格・報酬率反映 |

理由: 金銭が絡む昇格判定を初手から自動化するとロジックミスが報酬トラブルに直結するため、段階を踏む。

### 想定テーブル(イメージ)

```
agents (代理店)
- id, user_id, rank, referral_code

agent_sales (売上イベント)
- id, agent_id, buyer_user_id, amount, type(self/referral), created_at
```

### 決済フロー

```
Stripe決済 → 石高/ガチャ券購入 → ガチャ実行
```
(直接課金ではなく2段階にすることで会計処理・返金対応をシンプルにする)
